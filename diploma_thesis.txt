ABSTRACT
═══════════════════════════════════════════════════════════════════════════════

This thesis is devoted to the design, development, and evaluation of the
BOOKFLIX hybrid book recommendation system. The objective is to improve the
accuracy and relevance of personalised recommendations by transitioning from
the cascade architecture based on SVD and TF-IDF (developed in the 2023
coursework) to a Feature Combination hybrid that integrates Neural Collaborative
Filtering (NCF) implemented in PyTorch with semantic BERT embeddings and
VADER-based rating correction.

The thesis analyses the theoretical foundations of recommendation systems,
examines modern approaches to collaborative and content-based filtering, and
addresses the cold-start problem. A multi-layer software system was developed
on Django 3.2 with PostgreSQL, comprising: an NCF model training module, a
semantic embedding generation module via sentence-transformers
(all-MiniLM-L6-v2), a VADER sentiment analysis module, and a REST API with
web interface for user interaction.

Comparative evaluation confirmed the superiority of the new architecture:
RMSE decreased from 1.38 to 1.24, Precision@10 improved from 0.71 to 0.78,
and NDCG@10 reached 0.83 (a metric not previously computed). The dataset is
Book-Crossing (238,510 books, 1,031,178 ratings) enriched with goodbooks-10k.
The system is deployed on Render and validated through code quality analysis
(pylint 9.98/10, 45 passing tests).

Keywords: recommendation systems, neural collaborative filtering, BERT
embeddings, sentiment analysis, VADER, Feature Combination, hybrid methods,
cold-start, Django, deep learning.

Актуальність теми. Стрімке зростання обсягів цифрового контенту —
зокрема книжкових платформ, де каталоги нараховують мільйони найменувань, —
загострює проблему інформаційного перевантаження. Користувачеві дедалі
складніше самостійно орієнтуватися у пропозиціях і знаходити релевантні для
нього матеріали. Рекомендаційні системи (РС) є одним із ключових
інструментів вирішення цієї проблеми: вони аналізують поведінку та вподобання
користувачів і автоматично формують персоналізовану підбірку контенту.

Промислові системи класу Netflix, Amazon, Goodreads демонструють, що якість
рекомендацій безпосередньо впливає на залученість і монетизацію платформи.
За даними McKinsey, до 35% покупок Amazon генерується рекомендаційним
двигуном [1]. Однак більшість із цих систем базуються на складних ансамблях,
що потребують великих обчислювальних ресурсів. Для академічного та малого
промислового застосування актуальною залишається задача побудови
ефективної гібридної РС, яка об'єднує декілька сигналів без надмірного
ускладнення архітектури.

У попередньому дослідженні (курсова робота 2023, опублікована у вигляді
статті [2]) автором було розроблено систему BOOKFLIX на основі каскадного
гібриду: TF-IDF-ембедінги для контентно-орієнтованої фільтрації та SVD
(бібліотека scikit-surprise) для матричної факторизації. Метод TextBlob
застосовувався для аналізу тональності відгуків. Ця система показала RMSE=1,38
та Precision@5=0,71 на наборі даних Book-Crossing.

Разом із тим каскадний підхід має суттєві обмеження: семантичний компонент
не враховує навченого колаборативного сигналу при формуванні пулу
кандидатів, а лінійність SVD обмежує здатність моделі вловлювати нелінійні
взаємодії «користувач–книга». TextBlob, орієнтований на загальну англомовну
лексику, менш ефективний при аналізі книжкових відгуків порівняно зі
спеціалізованими інструментами на кшталт VADER.

Мета і задачі дослідження. Метою даної роботи є розробка та дослідження
вдосконаленої гібридної рекомендаційної системи BOOKFLIX, що забезпечує
вищу точність персоналізованих рекомендацій завдяки:
— заміні SVD на нейронну колаборативну фільтрацію (NCF) на основі PyTorch,
  здатну моделювати нелінійні взаємодії;
— переходу від TF-IDF до BERT-ембедінгів (sentence-transformers,
  all-MiniLM-L6-v2) для глибшого семантичного представлення книг;
— заміні TextBlob на VADER для точнішого аналізу тональності книжкових
  відгуків та м'якої корекції рейтингів;
— переходу від каскадної до Feature Combination архітектури, що об'єднує
  колаборативний та семантичний сигнали на мета-рівні.

1.1. Рекомендаційні системи: поняття та класифікація

Рекомендаційна система (РС) — це інформаційна система, що автоматично
прогнозує переваги користувача щодо певних елементів (товарів, фільмів,
книг, музики тощо) і формує персоналізовані підбірки на основі цих прогнозів.
Перші промислові РС з'явилися на початку 1990-х років у системах електронної
пошти GroupLens і комерційних платформах типу Amazon [3]. Відтоді область
значно еволюціонувала від простих евристичних підходів до складних ансамблів
із глибоким навчанням.

За механізмом роботи виділяють три основні класи рекомендаційних систем [4]:

1) Колаборативна фільтрація (Collaborative Filtering, CF) — ґрунтується
   на аналізі поведінки множини користувачів. Ключова ідея: якщо двоє
   користувачів мали схожі вподобання в минулому, вони, ймовірно, матимуть
   схожі вподобання у майбутньому. CF може бути пам'яттєвою (memory-based)
   або модельно-орієнтованою (model-based).

2) Контентно-орієнтована фільтрація (Content-Based Filtering, CBF) —
   ґрунтується на характеристиках самих елементів. Рекомендуються елементи,
   схожі за атрибутами до тих, що користувач раніше позитивно оцінив.
   Наприклад, якщо користувач читає детективи Агати Крісті, система
   рекомендуватиме інші детективи або книги цього автора.

3) Гібридні системи — поєднують CF і CBF з метою компенсації слабких
   сторін кожного підходу. Burke [5] виділяє сім стратегій гібридизації:
   зважена (weighted), перемикаюча (switching), змішана (mixed), каскадна
   (cascade), доповнювальна (augmented), метарівнева (meta-level) та
   комбінована на основі ознак (feature combination).

[РИС. 1.1 — Класифікація методів рекомендаційних систем (ієрархічна діаграма)]

Окремо виділяють системи на основі знань (knowledge-based), демографічні
системи та системи, що використовують контекст (context-aware). Проте для
задач рекомендації книг найбільш поширеними і результативними залишаються CF,
CBF та їх гібриди.

Ключові виклики, що постають перед рекомендаційними системами [6]:

— Проблема холодного старту (cold-start): труднощі з рекомендацією для
  нових користувачів або нових елементів, що не мають достатньої історії
  взаємодії.

— Проблема рідкісності (sparsity): матриця рейтингів «користувач–елемент»
  зазвичай надзвичайно розріджена — більшість пар не мають взаємодій.
  У наборі Book-Crossing рівень заповненості матриці становить менше 0,1%.

— Масштабованість: зі зростанням кількості користувачів та елементів
  обчислювальна складність наївних CF-алгоритмів стає неприйнятною.

— Різноманітність рекомендацій: уникнення «фільтрувальних бульбашок», де
  користувач отримує лише вузький спектр схожого контенту.

— Пояснюваність: здатність системи пояснити, чому рекомендований той чи
  інший елемент.

У контексті рекомендації книг виникають специфічні складнощі: книга є
багатовимірним артефактом (автор, жанр, стиль, тематика), читацькі вподобання
еволюціонують у часі, а рейтинги нерідко несуть суб'єктивний або упереджений
характер. Аналіз тональності відгуків дозволяє частково компенсувати цей
дисбаланс між числовим рейтингом та реальним сприйняттям книги.


1.2. Методи колаборативної фільтрації

Колаборативна фільтрація є одним з найдослідженіших напрямків у рекомендаційних
системах. Розрізняють два основних підходи: пам'яттєву (memory-based) та
модельну (model-based) CF [7].

Пам'яттєва CF (User-based та Item-based KNN). У пам'яттєвих методах схожість
обчислюється безпосередньо між рядками або стовпцями матриці рейтингів.
Наприклад, у User-based CF прогноз рейтингу користувача u для елемента i
обчислюється як:

    r̂(u,i) = r̄_u + (Σ_{v∈N(u)} sim(u,v)·(r(v,i) − r̄_v)) / Σ_{v∈N(u)} |sim(u,v)|  (1.1)

де N(u) — множина k найближчих сусідів користувача u, sim(u,v) — схожість
між користувачами (зазвичай косинусна або кореляція Пірсона).

Перевага пам'яттєвих методів — їх інтерпретованість і відсутність явного
навчального етапу. Недоліки — масштабованість O(|U|·|I|) та чутливість до
розрідженості матриці.

Матрична факторизація (Matrix Factorization, MF). Модельні методи CF
навчають латентні уявлення (embedding) користувачів та елементів. Найбільш
відомим є SVD-подібний підхід, що знайшов застосування у переможців конкурсу
Netflix Prize [8]. Базова модель:

    r̂(u,i) = μ + b_u + b_i + p_u^T · q_i                                    (1.2)

де μ — глобальне середнє, b_u та b_i — зміщення (bias) користувача та
елемента, p_u та q_i — латентні вектори розмірності d. Навчання відбувається
мінімізацією MSE з L2-регуляризацією:

    min_{p,q,b} Σ_{(u,i)∈K} (r(u,i) − r̂(u,i))² + λ(||p_u||² + ||q_i||² + b_u² + b_i²)  (1.3)

Бібліотека scikit-surprise пропонує готові реалізації SVD, SVD++, KNNBasic,
NMF тощо. У курсовій роботі 2023 року для системи BOOKFLIX використовувалась
саме SVD із scikit-surprise.

Нейронна колаборативна фільтрація (NCF). He et al. [9] запропонували замінити
внутрішній добуток p_u^T·q_i в MF на нелінійний MLP, що дозволяє моделювати
складніші взаємодії:

    ŷ(u,i) = f(p_u, q_i | θ)                                                 (1.4)

де f — довільна нелінійна функція, що параметризується нейронною мережею θ.
Детальніший розгляд NCF наведено у розділі 2.3.

Порівняння методів CF за характеристиками наведено у таблиці 1.1.

[ТАБЛИЦЯ 1.1 — Порівняння методів колаборативної фільтрації]

| Метод          | Масштаб. | Точність | Холодний старт | Інтерпрет. |
|----------------|----------|----------|----------------|------------|

1.3. Методи контентно-орієнтованої фільтрації

Контентно-орієнтована фільтрація (CBF) формує профіль інтересів користувача
на основі ознак елементів, які він позитивно оцінив, і рекомендує нові елементи
з максимальною схожістю до цього профілю. Для текстових елементів (книги,
статті, новини) центральне місце займає текстове представлення.

TF-IDF та лексичні ембедінги. Класичний підхід — векторизація текстових описів
(назва, автор, жанр, анотація) за допомогою TF-IDF. Вага терміна t в документі
d:

    tfidf(t,d) = tf(t,d) · log(N / df(t))                                    (1.5)

де tf(t,d) — частота терміна, N — загальна кількість документів, df(t) —
кількість документів, що містять термін t.

Зниження розмірності (Latent Semantic Analysis, LSA) через Truncated SVD
дозволяє отримати компактне семантичне представлення розмірності ~100. У
курсовій роботі 2023 використовувались TF-IDF + LSA для побудови ембедінгів
книг.

Word2Vec, GloVe та fastText. Методи навчання щільних векторних представлень
слів на великих корпусах. Ембедінги документів отримуються усередненням
ембедінгів слів. Ці підходи кращі за TF-IDF для вловлення синонімії та
семантичної близькості, але не враховують контекст позиції слів у реченні.

BERT та трансформерні моделі. Дев'ять і Варданян (Devlin et al., 2019) [10]
запропонували двоспрямований трансформер BERT, що генерує контекстуально
залежні ембедінги. На відміну від Word2Vec, одне і те саме слово може мати
різні ембедінги залежно від контексту.

Для задачі семантичного пошуку та порівняння речень оптимальним є
sentence-transformers [11] — бібліотека, що навчає BERT на парних задачах
(Siamese networks) для отримання семантично зв'язних sentence-level ембедінгів.
Модель all-MiniLM-L6-v2 є ефективним і компактним варіантом із 384-вимірними
ембедінгами при відмінному балансі якість/швидкість.

Подібність між документами обчислюється через косинусну схожість:

    cos(a, b) = (a · b) / (||a|| · ||b||)                                    (1.6)

Для L2-нормалізованих ембедінгів, як у all-MiniLM-L6-v2, косинусна схожість
еквівалентна скалярному добутку: cos(â, b̂) = â · b̂.


1.4. Гібридні підходи та їх порівняння

Гібридні рекомендаційні системи поєднують CF та CBF для подолання їх взаємних
недоліків: CF погано справляється з холодним стартом для нових елементів,
а CBF обмежений у здатності знаходити «нове та неочікуване». Burke [5]
систематизує підходи до гібридизації:

Зважена гібридизація (Weighted). Об'єднує бали кількох систем лінійно:
    score(u,i) = α·score_CF(u,i) + (1−α)·score_CBF(u,i)                    (1.7)

Каскадна гібридизація (Cascade). Одна система формує пул кандидатів,
інша ранжує їх. У BOOKFLIX 2023: CBF (TF-IDF) → пул → SVD (ранжування).

Feature Combination (FC). Ознаки обох систем об'єднуються в одному векторі
або нормалізовані бали зважуються до надання рекомендацій. Ця стратегія
не потребує двох окремих підсистем — сигнали комбінуються на мета-рівні.

[РИС. 1.2 — Порівняння каскадної та Feature Combination гібридних архітектур]

Дослідження показують, що Feature Combination, як правило, перевершує каскадний
підхід: помилки не «накопичуються» між стадіями, а обидва сигнали враховуються
рівноправно при ранжуванні [12].

| Характеристика          | Cascade (BOOKFLIX 2023) | FC (BOOKFLIX 2025) |
|-------------------------|-------------------------|--------------------|
| Порядок комбінування    | Послідовний             | Паралельний        |
| Вплив помилок 1-ї стадії| Посилюється             | Відсутній          |
| Гнучкість ваг           | Відсутня                | W_NCF=0,6; W_SEM=0,4 |
| Холодний старт          | Часткова підтримка      | Повна підтримка    |

1.5. Нейронні мережі в рекомендаційних системах

Deep Learning методи активно впроваджуються в рекомендаційні системи з 2016
року. Zhang et al. [13] класифікують DL-підходи для РС:

— MLP та автоенкодери (NCF, AutoRec);
— Рекурентні мережі (RNN, LSTM, GRU) для послідовних рекомендацій;
— Згорткові мережі (CNN) для обробки тексту та зображень;
— Графові нейронні мережі (GNN) для моделювання соціальних зв'язків;
— Трансформери (BERT4Rec, SASRec) для послідовного та семантичного аналізу.

NCF (He et al., 2017) [9] — базова нейронна модель CF. Ключова ідея: замість
лінійного внутрішнього добутку в MF використовувати MLP для нелінійного
відображення конкатенованих ембедінгів користувача та елемента:

    ŷ(u,i) = MLP([e_u ; e_i])                                                (1.8)

де [·;·] — конкатенація, e_u та e_i — навчені ембедінги. На наборі MovieLens-1M
NCF перевершував SVD на 4,4% за HR@10 та 5,5% за NDCG@10 [9].

NeuMF (Neural Matrix Factorization) — об'єднує MF та NCF через late fusion:
    ŷ = σ(h^T [GMF output; MLP output])                                      (1.9)

Для задач рекомендації книг Chaban & Pylypenko [2] показали, що навіть
помірна NCF-модель перевершує SVD завдяки здатності моделювати нелінійні
взаємодії, характерні для читацьких уподобань.

Трансформери та LLM. BERT4Rec [14] та SASRec [15] використовують
self-attention для моделювання послідовностей взаємодій, що особливо ефективно
при повторних відвідуваннях та читанні серій книг. Проте ці моделі потребують
значно більших обчислювальних ресурсів. Для BOOKFLIX було обрано NCF як
більш ефективний для наявного набору даних компроміс між якістю та
продуктивністю.


1.6. Аналіз попередньої системи (курсова робота 2023)

Система BOOKFLIX, розроблена у 2023 році [2], реалізувала каскадний гібрид
на наборі даних Book-Crossing із наступною архітектурою:

Архітектура 2023 року:
1. Попередня обробка: очищення даних Book-Crossing, видалення ISBN у форматі
   наукової нотації, нормалізація рейтингів.
2. Контентна частина: TF-IDF векторизація полів title+author+publisher з
   подальшим LSA (TruncatedSVD, 100 компонент). Косинусна схожість між
   книгами та профілем користувача формувала пул кандидатів.
3. Колаборативна частина: SVD (scikit-surprise, n_factors=100, n_epochs=20,
   lr_all=0.005, reg_all=0.02) для матричної факторизації та ранжування
   пулу.
4. Аналіз тональності: TextBlob для розрахунку полярності відгуків із CSV
   (bookflix/sentiment_clean.py). Перетворення полярності в категорії
   Positive/Negative/Neutral.
5. Веб-інтерфейс: Django із сесійним профілюванням («смак» користувача
   на основі оцінок 5 книг).

[РИС. 1.3 — Архітектура BOOKFLIX 2023 (каскадний гібрид)]

Метрики системи 2023 (на тестовій вибірці Book-Crossing):
— RMSE = 1,38 (SVD)
— Precision@5 = 0,71 (без TextBlob-корекції)
— Precision@5 = 0,74 (із TextBlob-корекцією рейтингів)
— NDCG не вимірювався

Виявлені слабкі місця:
— Лінійність SVD не дозволяє моделювати складні нелінійні взаємодії між
  користувачами та книгами (особливо для читачів із нестандартними смаками).
— TF-IDF ігнорує синоніми та паронімії: «детектив» і «crime fiction» не
  будуть схожими для лексичної моделі, але схожими для семантичної.
— Каскадна помилка: якщо CBF не включила релевантну книгу до пулу кандидатів,
  SVD не матиме шансу її ранжувати.
— TextBlob навчений на загальних текстах, а не на книжкових відгуках, що
  обмежує точність аналізу тональності порівняно з VADER [16].
— Відсутність метрики NDCG не дозволяла оцінити якість порядку видачі.
— scikit-surprise не підтримує GPU-прискорення, що обмежує масштабованість.

Ці обмеження стали відправною точкою для розробки нової архітектури BOOKFLIX
2025, описаної в розділі 2.


1.7. Постановка задачі

На підставі аналізу предметної галузі та виявлених обмежень попередньої системи
формулюємо задачу дослідження:

Розробити гібридну рекомендаційну систему BOOKFLIX, що забезпечує:

1. Підвищення точності колаборативної фільтрації через заміну SVD на NCF
   (PyTorch MLP), здатну моделювати нелінійні взаємодії «користувач–книга».

2. Покращення якості контентного представлення через заміну TF-IDF+LSA на
   BERT-ембедінги (sentence-transformers, all-MiniLM-L6-v2, 384 вим.), що
   вловлюють глибокий семантичний сенс.

3. Підвищення якості навчальних даних через VADER-корекцію рейтингів,
   що зменшує шум у числових оцінках завдяки врахуванню тональності відгуків.

4. Перехід від каскадної до Feature Combination гібридної архітектури, де
   обидва сигнали (NCF та BERT) об'єднуються на мета-рівні перед ранжуванням.

5. Вирішення проблеми холодного старту через механізм інференсу ембедінгу
   нового користувача без перенавчання моделі.

6. Розширення набору метрик оцінювання системи: додавання NDCG@K на додаток
   до RMSE та Precision@K.

7. Забезпечення промислової якості реалізації: pylint ≥ 9,9/10, повне
   тестове покриття (≥ 45 тестів), розгортання на Render.

Математична постановка задачі рекомендації полягає у наступному. Нехай U —
множина користувачів, I — множина книг, R ⊂ U×I×[1,10] — матриця рейтингів.
Задача рекомендації: для заданого користувача u ∈ U знайти впорядкований
список книг TopK(u) ⊂ I\I_u, де I_u = {i : (u,i,·) ∈ R} — книги, вже
прочитані користувачем, що максимізує:

    TopK(u) = argmax_{i∉I_u, |S|=K} Σ_{i∈S} score(u, i)                    (1.9)

де score(u,i) = W_NCF·ncf_norm(u,i) + W_SEM·sem_norm(u,i) — комбінований бал
системи BOOKFLIX 2025.

2.1. Загальна архітектура системи

Система BOOKFLIX 2025 побудована як багаторівневий веб-застосунок на Django 3.2.
Архітектура системи представлена на рисунку 2.1 і включає наступні рівні:

[РИС. 2.1 — Загальна архітектура системи BOOKFLIX (UML-компонентна діаграма)]

Рівень даних (Data Layer):
— PostgreSQL (Render) / SQLite (локальна розробка) — основна реляційна СУБД
  з трьома ключовими моделями: Book, User, Rating.
— CSV-файли (Book-Crossing + goodbooks-10k) — джерело даних для навчання.
— GitHub Releases — зберігання великих CSV-файлів (~200 MB), що завантажуються
  при розгортанні.

Рівень ML (Machine Learning Layer) — пакет bookflix/ml/:
— ncf.py       — NCF-модель (PyTorch)
— embeddings.py — BERTEmbedder та LSAEmbedder (запасний)
— hybrid.py    — Feature Combination та cascade_recommend (порівняння)
— evaluation.py — метрики RMSE, Precision@K, NDCG@K
— sentiment.py  — VADER-корекція рейтингів

Рівень бізнес-логіки (Service Layer):
— recommendation_algorithms.py — уніфікована точка входу, завантаження даних,

Попередня обробка включала:

1. Виправлення ISBN. Частина ISBN зберігалась у вигляді числа з плаваючою

2. Видалення неявних рейтингів (book_rating == 0). Такі записи не несуть
   корисного сигналу для навчання з вчителем.

3. Нормалізація полів. Усі рядкові поля в Django-моделях змінено з CharField
   на TextField для сумісності з PostgreSQL (відсутність обмеження довжини).

4. VADER-корекція. Для книг, що мають відгуки в supplementary CSV-файлі,
   числовий рейтинг коригується за формулою (описана у 2.5).

Розподіл між навчальним і тестовим наборами виконується через стратифіковане
розбиття per-rating 80/20, реалізоване у функції train_test_split:

Фрагмент коду 2.1 — Функція train_test_split (recommendation_algorithms.py):

    def train_test_split(df, test_ratio=0.2, seed=42):
        rng = np.random.default_rng(seed)
        train_idx, test_idx = [], []
        for _, group in df.groupby("user_id"):
            idx = group.index.tolist()
            if len(idx) == 1:
                train_idx.extend(idx)
                continue
            n_test = max(1, int(len(idx) * test_ratio))
            chosen = rng.choice(idx, size=n_test, replace=False)
            test_set = set(chosen)
            test_idx.extend(chosen)
            train_idx.extend(i for i in idx if i not in test_set)
        return df.loc[train_idx], df.loc[test_idx]

Ключова властивість цієї реалізації: кожен користувач з ≥ 2 рейтингами
гарантовано представлений у навчальному наборі. Для користувачів з єдиним
рейтингом він повністю переходить до train. Це важливо для уникнення проблеми
cold-start під час оцінювання.


2.3. Нейронна колаборативна фільтрація (NCF)

NCF-модель реалізована у файлі bookflix/ml/ncf.py на основі PyTorch та
відтворює архітектуру He et al. [9] з модифікаціями для підтримки
холодного старту.

Архітектура моделі. NCF складається з двох шарів ембедінгів та стека MLP:

    e_u = Embedding(num_users+1, d)   # ембедінг користувача
    e_i = Embedding(num_items+1, d)   # ембедінг книги
    x   = concat(e_u, e_i)           # (2d,)
    ŷ   = MLP(x)                      # → скаляр

де d = 32 (embedding_dim), стек MLP: Linear(64)→ReLU→Dropout(0.2)→
Linear(32)→ReLU→Dropout(0.2)→Linear(16)→ReLU→Dropout(0.2)→Linear(1).

[РИС. 2.4 — Архітектура NCF-моделі (діаграма нейронної мережі)]

Гіперпараметри, обрані після grid-search:
— embedding_dim = 32
— hidden_layers = [64, 32, 16]
— dropout = 0.2
— optimizer: Adam, lr=1e-3
— loss: MSELoss (для рейтингового прогнозування)
— epochs: 20, batch_size: 512

Фрагмент коду 2.2 — Клас NCF (bookflix/ml/ncf.py):

    class NCF(nn.Module):
        def __init__(self, num_users, num_items, embedding_dim=32,
                     hidden_layers=None, dropout=0.2):
            super().__init__()
            if hidden_layers is None:
                hidden_layers = [64, 32, 16]
            self.user_embedding = nn.Embedding(
                num_users + 1, embedding_dim, padding_idx=0)
            self.item_embedding = nn.Embedding(
                num_items + 1, embedding_dim, padding_idx=0)
            layers = []
            in_dim = embedding_dim * 2
            for out_dim in hidden_layers:
                layers += [nn.Linear(in_dim, out_dim),
                           nn.ReLU(), nn.Dropout(dropout)]
                in_dim = out_dim
            layers.append(nn.Linear(in_dim, 1))
            self.mlp = nn.Sequential(*layers)
            nn.init.normal_(self.user_embedding.weight, std=0.01)
            nn.init.normal_(self.item_embedding.weight, std=0.01)

        def forward(self, user_ids, item_ids):
            u = self.user_embedding(user_ids)
            v = self.item_embedding(item_ids)
            x = torch.cat([u, v], dim=-1)
            return self.mlp(x).squeeze(-1)

Ініціалізація ваг ембедінгів нормальним розподілом N(0, 0.01) забезпечує
стабільний початок навчання без домінування жодного з вимірів [9].

Padding_idx=0 зарезервований для нульового ембедінгу (невідомий
користувач/книга), що є технічним рішенням для підтримки холодного старту.

Процес навчання. Навчання виконується в recommendation_algorithms.py:
пари (user_id, item_id, rating) завантажуються з бази Django або CSV,
нормалізуються до діапазону [0,1], і модель навчається мінімізувати MSE
між прогнозованими та реальними рейтингами.

[РИС. 2.5 — Крива навчання NCF (train loss / val loss по епохах)]

Збережений стан моделі (state_dict) зберігається як .pt файл для подальшого

2.4. Семантичні BERT-ембедінги

Модуль bookflix/ml/embeddings.py реалізує два варіанти ембедерів:
BERTEmbedder (основний) та LSAEmbedder (запасний, для середовищ без GPU).

BERTEmbedder використовує модель sentence-transformers all-MiniLM-L6-v2 [11]:
— 6 трансформерних шарів (замість 12 у base BERT)
— 384-вимірні ембедінги
— Навчена на задачах semantic textual similarity (STS)
— Швидкість: ~14,200 речень/сек на CPU

Вхідний текст формується шляхом конкатенації полів: «{title}. By {author}»,
що дозволяє одночасно враховувати назву та автора книги.

Фрагмент коду 2.3 — BERTEmbedder (bookflix/ml/embeddings.py):

    class BERTEmbedder:
        MODEL_NAME = "all-MiniLM-L6-v2"

        def __init__(self):
            from sentence_transformers import SentenceTransformer
            self.model = SentenceTransformer(self.MODEL_NAME)
            self.embeddings = None
            self.isbns = []
            self.isbn_to_idx = {}

        def fit(self, books_df):
            sentences = (
                books_df["title"].fillna("") + ". By " +
                books_df["author"].fillna("")
            ).tolist()
            from sklearn.preprocessing import normalize
            raw = self.model.encode(
                sentences, show_progress_bar=True, batch_size=64)
            self.embeddings = normalize(raw).astype(np.float32)
            self.isbns = books_df["isbn"].tolist()
            self.isbn_to_idx = {isbn: i for i, isbn in enumerate(self.isbns)}

        def get(self, isbn):
            idx = self.isbn_to_idx.get(isbn)
            if idx is None:
                return np.zeros(384, dtype=np.float32)
            return self.embeddings[idx]

        def build_user_profile(self, liked_isbns):
            vecs = [self.get(isbn) for isbn in liked_isbns
                    if isbn in self.isbn_to_idx]
            if not vecs:
                return None
            profile = np.mean(vecs, axis=0)
            norm = np.linalg.norm(profile)
            return profile / norm if norm > 0 else profile

L2-нормалізація (normalize з scikit-learn) дозволяє обчислювати косинусну
схожість через простий скалярний добуток, що є значно ефективнішим:

    cos_sim(a, b) = â · b̂,   де â = a / ||a||                              (2.1)

Профіль користувача будується як середній вектор ембедінгів книг, що
користувач позитивно оцінив (rating ≥ RELEVANCE_THRESHOLD = 7), з наступною
нормалізацією. Такий підхід є різновидом «Rocchio algorithm» для
контентно-орієнтованих систем [19].

LSAEmbedder (запасний варіант) використовує TF-IDF + TruncatedSVD (100 вим.)
та cosine_similarity зі scikit-learn. Він застосовується автоматично, якщо
sentence-transformers недоступний у середовищі виконання.

2.5. Аналіз тональності VADER та корекція рейтингів

Модуль bookflix/ml/sentiment.py реалізує корекцію рейтингів на основі
аналізу тональності відгуків. Використовується VADER (Valence Aware Dictionary
and sEntiment Reasoner) [20] як основний інструмент із TextBlob як запасним.

Вибір VADER замість TextBlob обґрунтований тим, що:
1. VADER оптимізований для коротких оцінних текстів (відгуки, соцмережі).
2. VADER враховує пунктуацію (!!!), капіталізацію (GREAT!) та slang.
3. На наборах книжкових відгуків VADER показує вищу відповідність людській
   оцінці тональності порівняно з TextBlob [16].

VADER повертає compound score ∈ [-1, 1], де -1 = гранично негативний,
0 = нейтральний, +1 = гранично позитивний.

Формула корекції рейтингу:

    r̃ = clip(r + α · compound, 1, 10)                                       (2.2)

де r — оригінальний рейтинг (1–10), compound — середній VADER-compound по
всіх відгуках до книги, α = 0,5 — коефіцієнт корекції.

Приклад: книга з рейтингом 7 та compound = 0,6 отримає скоригований рейтинг
7 + 0,5 · 0,6 = 7,3 (кліп в [1,10] не спрацьовує).

Фрагмент коду 2.4 — Функція apply_sentiment_correction (bookflix/ml/sentiment.py):

    ALPHA = 0.5

    def apply_sentiment_correction(ratings_df, sentiment_map, alpha=ALPHA):
        isbn_col = "book__isbn"
        ratings_df = ratings_df.copy()
        ratings_df["sentiment_score"] = (
            ratings_df[isbn_col].map(sentiment_map).fillna(0.0)
        )
        ratings_df["adjusted_rating"] = (
            ratings_df["book_rating"] + alpha * ratings_df["sentiment_score"]
        ).clip(1, 10)
        return ratings_df

Коефіцієнт α = 0,5 обраний на основі абляційного аналізу (розділ 3.5):
нижчі значення (0,1–0,3) мали мінімальний вплив, вищі (0,7–1,0) призводили
до надмірної корекції та погіршення RMSE.

Функція build_book_sentiment_map будує словник {isbn: mean_compound} з CSV
з відгуками, що передається до apply_sentiment_correction перед навчанням
NCF.

[ТАБЛИЦЯ 2.2 — Розподіл тональності відгуків у наборі Book-Crossing]

| Категорія  | Діапазон compound | Кількість книг (приблиз.) |
|------------|-------------------|---------------------------|
| Позитивна  | ≥ 0,05            | 68%                       |
| Нейтральна | (-0,05; 0,05)     | 19%                       |
| Негативна  | ≤ -0,05           | 13%                       |

2.6. Feature Combination гібридна модель

Модуль bookflix/ml/hybrid.py реалізує два підходи: feature_combination_recommend
(основний, BOOKFLIX 2025) та cascade_recommend (застарілий, BOOKFLIX 2023,
збережений для порівняння).

Принцип Feature Combination полягає в об'єднанні нормалізованих до [0,1]
балів NCF та семантичної схожості BERT перед ранжуванням:

    score(u,i) = W_NCF · ncf_norm(u,i) + W_SEM · sem_norm(u,i)             (2.3)

де W_NCF = 0,6, W_SEM = 0,4, нормалізація:

    x_norm = (x − x_min) / (x_max − x_min)                                  (2.4)

по всьому пулу кандидатів.

Обґрунтування ваг W_NCF=0,6, W_SEM=0,4. Нейронна колаборативна фільтрація
моделює персоналізований сигнал (що саме цей користувач любить), тоді як
BERT відображає семантичну близькість до вподобань. CF-сигнал вважається
більш персоналізованим, тому отримує більшу вагу. Оптимальність цих ваг
підтверджена абляційним аналізом у розділі 3.5.

Фрагмент коду 2.5 — feature_combination_recommend (bookflix/ml/hybrid.py):

    W_NCF = 0.6
    W_SEM = 0.4

    def feature_combination_recommend(
        user_id, candidate_isbns, embedder,
        rated_isbns, ncf_score_fn, top_n=10,
        w_ncf=W_NCF, w_sem=W_SEM,
    ):
        if not candidate_isbns:
            return []
        user_profile = embedder.build_user_profile(rated_isbns)
        if user_profile is None:
            sem_scores = np.zeros(len(candidate_isbns))
        else:
            book_vecs = np.stack(
                [embedder.get(isbn) for isbn in candidate_isbns])
            sem_scores = book_vecs @ user_profile
        ncf_raw = np.array([
            ncf_score_fn(user_id, isbn) or 5.0
            for isbn in candidate_isbns
        ])
        ncf_norm = _normalise(ncf_raw)
        sem_norm = _normalise(sem_scores)
        combined = w_ncf * ncf_norm + w_sem * sem_norm
        ranked_idx = np.argsort(combined)[::-1][:top_n]
        return [
            {"isbn": candidate_isbns[i],
             "ncf_score": round(float(ncf_raw[i]), 3),
             "sem_score": round(float(sem_scores[i]), 3),
             "score": round(float(combined[i]), 3)}
            for i in ranked_idx
        ]

Пул кандидатів формується зі всіх книг, що користувач ще не оцінив.
Для ефективності обмежується до 2000–5000 книг з найвищим goodreads_rating
або останніх за ISBN.

[РИС. 2.8 — Схема Feature Combination: паралельне обчислення NCF та BERT балів
             з наступним зваженим об'єднанням]

Порівняння з каскадним підходом (cascade_recommend):
У cascade_recommend CBF формує пул із 40 кандидатів (за косинусною схожістю),
після чого NCF їх ранжує. Проблема: якщо релевантна книга не потрапила в
топ-40 за CBF, вона взагалі не з'явиться у рекомендаціях. Feature Combination
уникає цієї проблеми, розглядаючи всіх кандидатів одночасно.


2.7. Вирішення проблеми холодного старту

Проблема холодного старту (cold-start) для нових користувачів є однією з
центральних у рекомендаційних системах. У BOOKFLIX 2025 вирішення здійснюється
через метод infer_user_embedding класу NCF.

Ідея: якщо новий користувач оцінив k книг у сесії (але не присутній у
навчальних даних NCF), його псевдо-ембедінг обчислюється як зважене середнє
ембедінгів оцінених книг, де вагами є відносні рейтинги:

    ê_u = Σ_{(i,r) ∈ session} (r / Σr') · e_i                              (2.5)

де e_i — ембедінг книги i, r — рейтинг, Σr' — сума рейтингів у сесії.

Фрагмент коду 2.6 — infer_user_embedding (bookflix/ml/ncf.py):

    def infer_user_embedding(self, item_ratings):
        if not item_ratings:
            emb_dim = self.user_embedding.embedding_dim
            return torch.zeros(1, emb_dim)
        with torch.no_grad():
            item_idxs = torch.tensor(
                [idx for idx, _ in item_ratings], dtype=torch.long)
            ratings = torch.tensor(
                [float(r) for _, r in item_ratings], dtype=torch.float32)
            item_embs = self.item_embedding(item_idxs)
            weights = (ratings / ratings.sum()).unsqueeze(1)
            user_emb = (item_embs * weights).sum(dim=0, keepdim=True)
        return user_emb

Метод forward_with_inferred_embedding використовує цей ембедінг замість
навченого для прогнозування рейтингів нових книг:

    def forward_with_inferred_embedding(self, user_emb, item_ids):
        item_embs = self.item_embedding(item_ids)
        user_embs = user_emb.expand(item_ids.shape[0], -1)
        x = torch.cat([user_embs, item_embs], dim=-1)
        return self.mlp(x).squeeze(-1)

Перевага цього підходу — відсутність необхідності перенавчання: нові
користувачі одразу отримують персоналізовані рекомендації після оцінювання
5–10 книг у сесії (сторінка «Мій смак»).

[РИС. 2.9 — Схема вирішення cold-start: сесійний профіль → infer_user_embedding
             → NCF forward_with_inferred_embedding → Feature Combination]

Для семантичного компонента проблеми холодного старту не виникає: профіль
будується з ембедінгів книг, які вже оцінено, а build_user_profile обчислює
середній ембедінг незалежно від наявності користувача в БД.

3.1. Метрики оцінювання рекомендаційних систем

Для комплексного оцінювання якості рекомендацій використовуються три метрики,
реалізовані у bookflix/ml/evaluation.py.

RMSE (Root Mean Square Error) — метрика точності прогнозування рейтингів:

    RMSE = √(1/n · Σ_{i=1}^{n} (r_i − r̂_i)²)                             (3.1)

де r_i — реальний рейтинг, r̂_i — прогнозований рейтинг. Нижче RMSE —
кращий рейтинговий прогноз. Метрика не враховує порядок рекомендацій.

Precision@K — частка релевантних елементів серед K рекомендованих:

    Precision@K = (1/|U|) · Σ_{u∈U} |TopK(u) ∩ Rel(u)| / K               (3.2)

де Rel(u) = {i : r(u,i) ≥ RELEVANCE_THRESHOLD} — множина релевантних книг
(тих, що user оцінив на ≥ 7), TopK(u) — топ-K рекомендацій для u.

NDCG@K (Normalized Discounted Cumulative Gain) — метрика якості ранжування,
що враховує позицію релевантного елемента у списку:

    DCG@K = Σ_{k=1}^{K} rel_k / log₂(k+1)                                  (3.3)
    NDCG@K = DCG@K / IDCG@K                                                  (3.4)

де IDCG@K — DCG ідеального ранжування (всі релевантні елементи на першх
позиціях). NDCG ∈ [0,1], де 1 = ідеальне ранжування.

Порівняно з системою 2023, у новій системі:
— RMSE залишається відповідним показником точності прогнозу SVD vs NCF.
— Precision@K тепер обчислюється при K=10 (проти K=5 у 2023), що більш
  реалістично для книжкових рекомендацій.
— NDCG@K вперше введено до системи BOOKFLIX для оцінки якості порядку видачі.

Реалізація метрик (bookflix/ml/evaluation.py):

    RELEVANCE_THRESHOLD = 7

    def compute_rmse(actuals, predictions):
        return float(np.sqrt(mean_squared_error(actuals, predictions)))

    def compute_precision_at_k(user_recommendations, user_relevant_items, k=10):
        scores = []
        for uid, recs in user_recommendations.items():
            relevant = user_relevant_items.get(uid, set())
            if not relevant:
                continue
            hits = sum(1 for isbn in recs[:k] if isbn in relevant)
            scores.append(hits / k)
        return float(np.mean(scores)) if scores else 0.0

    def compute_ndcg_at_k(user_recommendations, user_relevant_items, k=10):
        scores = []
        for uid, recs in user_recommendations.items():
            relevant = user_relevant_items.get(uid, set())
            if not relevant:
                continue
            gains = [1 if isbn in relevant else 0 for isbn in recs[:k]]
            dcg = _dcg_at_k(gains, k)
            ideal = _dcg_at_k([1]*min(len(relevant), k), k)
            if ideal > 0:
                scores.append(dcg / ideal)
        return float(np.mean(scores)) if scores else 0.0

3.4. Порівняльний аналіз архітектур

Порівняльний аналіз охоплює чотири конфігурації системи:

             для чотирьох конфігурацій]

Ключові висновки з порівняльного аналізу:

1. NCF vs SVD (A→B): RMSE покращився з 1,38 до 1,31 (-5,1%), що підтверджує
   перевагу нелінійного MLP над лінійним SVD для моделювання взаємодій.

2. Feature Combination vs NCF-only (B→C): RMSE покращився до 1,26 (-3,8%),
   Precision@10 зріс з 0,72 до 0,75. Додавання семантичного BERT-компонента
   покращує ранжування завдяки врахуванню контентної подібності.

3. VADER-корекція (C→D): RMSE знизився до 1,24 (-1,6%), Precision@10 зріс
   до 0,78, NDCG@10 — до 0,83. VADER-корекція зменшує шум у навчальних
   даних, що дає помірне, але стійке покращення всіх метрик.

4. Підсумковий приріст (A→D): RMSE зменшився на 10,1%, Precision@10
   збільшився на 9,9%.

В ході виконання роботи отримано такі результати:

1. Проведено комплексний аналіз методів рекомендаційних систем: колаборативної

   Встановлено, що Feature Combination у поєднанні з нейронними методами
   забезпечує вищу якість рекомендацій порівняно з каскадними підходами.

2. Розроблено та реалізовано NCF-модель (PyTorch MLP, embedding_dim=32,
   hidden=[64,32,16], dropout=0.2), що замінила SVD із scikit-surprise.

3. Реалізовано модуль BERT-ембедінгів на основі sentence-transformers
   (all-MiniLM-L6-v2, 384 вим.) із L2-нормалізацією та побудовою профілю
   користувача через середній ембедінг позитивно оцінених книг.

4. Розроблено механізм VADER-корекції рейтингів (adjusted_rating =
   clip(r + 0.5·compound, 1, 10)), що знизив RMSE на 1,6% порівняно з
   Feature Combination без корекції тональності.

5. Реалізовано Feature Combination гібридну модель з вагами W_NCF=0,6,
   W_SEM=0,4, що об'єднує нормалізовані бали NCF та BERT. Порівняльний
   аналіз підтвердив переваги над каскадним підходом.

6. Вирішено проблему холодного старту через метод infer_user_embedding,
   що обчислює псевдо-ембедінг нового користувача як зважене середнє
   ембедінгів оцінених книг без перенавчання моделі.

7. Розширено набір метрик оцінювання: до RMSE та Precision@K додано NDCG@K,
   що дозволяє оцінювати якість порядку видачі рекомендацій. Фінальні
   метрики системи: RMSE=1,24, Precision@10=0,78, NDCG@10=0,83.

8. Розроблено повноцінний Django-застосунок із REST API, веб-інтерфейсом
   та розгортанням на Render. Якість коду: pylint 9,98/10, 45 тестів, flake8
   та isort — нульові порушення.

А.1. Модель NCF (bookflix/ml/ncf.py)

"""
Neural Collaborative Filtering (NCF) — Section 2.3 of thesis.
"""

import torch
import torch.nn as nn


class NCF(nn.Module):
    def __init__(self, num_users, num_items, embedding_dim=32,
                 hidden_layers=None, dropout=0.2):
        super().__init__()
        if hidden_layers is None:
            hidden_layers = [64, 32, 16]
        self.user_embedding = nn.Embedding(
            num_users + 1, embedding_dim, padding_idx=0)
        self.item_embedding = nn.Embedding(
            num_items + 1, embedding_dim, padding_idx=0)
        layers = []
        in_dim = embedding_dim * 2
        for out_dim in hidden_layers:
            layers += [nn.Linear(in_dim, out_dim),
                       nn.ReLU(), nn.Dropout(dropout)]
            in_dim = out_dim
        layers.append(nn.Linear(in_dim, 1))
        self.mlp = nn.Sequential(*layers)
        nn.init.normal_(self.user_embedding.weight, std=0.01)
        nn.init.normal_(self.item_embedding.weight, std=0.01)

    def forward(self, user_ids: torch.Tensor, item_ids: torch.Tensor):
        u = self.user_embedding(user_ids)
        v = self.item_embedding(item_ids)
        x = torch.cat([u, v], dim=-1)
        return self.mlp(x).squeeze(-1)

    def infer_user_embedding(self, item_ratings: list) -> torch.Tensor:
        if not item_ratings:
            return torch.zeros(1, self.user_embedding.embedding_dim)
        with torch.no_grad():
            item_idxs = torch.tensor(
                [idx for idx, _ in item_ratings], dtype=torch.long)
            ratings = torch.tensor(
                [float(r) for _, r in item_ratings], dtype=torch.float32)
            item_embs = self.item_embedding(item_idxs)
            weights = (ratings / ratings.sum()).unsqueeze(1)
            return (item_embs * weights).sum(dim=0, keepdim=True)

    def forward_with_inferred_embedding(self, user_emb, item_ids):
        item_embs = self.item_embedding(item_ids)
        user_embs = user_emb.expand(item_ids.shape[0], -1)
        x = torch.cat([user_embs, item_embs], dim=-1)
        return self.mlp(x).squeeze(-1)

    def get_user_embedding(self, user_idx: int) -> torch.Tensor:
        with torch.no_grad():
            return self.user_embedding(torch.tensor([user_idx]))

    def get_item_embedding(self, item_idx: int) -> torch.Tensor:
        with torch.no_grad():
            return self.item_embedding(torch.tensor([item_idx]))


А.2. Гібридна модель — Feature Combination (bookflix/ml/hybrid.py)

import numpy as np

W_NCF = 0.6
W_SEM = 0.4


def _normalise(arr: np.ndarray) -> np.ndarray:
    lo, hi = arr.min(), arr.max()
    if hi == lo:
        return np.zeros_like(arr)
    return (arr - lo) / (hi - lo)


def feature_combination_recommend(
    user_id, candidate_isbns, embedder,
    rated_isbns, ncf_score_fn, top_n=10,
    w_ncf=W_NCF, w_sem=W_SEM,
):
    if not candidate_isbns:
        return []
    user_profile = embedder.build_user_profile(rated_isbns)
    if user_profile is None:
        sem_scores = np.zeros(len(candidate_isbns))
    else:
        book_vecs = np.stack(
            [embedder.get(isbn) for isbn in candidate_isbns])
        sem_scores = book_vecs @ user_profile
    ncf_raw = np.array([
        ncf_score_fn(user_id, isbn) or 5.0
        for isbn in candidate_isbns
    ])
    ncf_norm = _normalise(ncf_raw)
    sem_norm = _normalise(sem_scores)
    combined = w_ncf * ncf_norm + w_sem * sem_norm
    ranked_idx = np.argsort(combined)[::-1][:top_n]
    return [
        {"isbn": candidate_isbns[i],
         "ncf_score": round(float(ncf_raw[i]), 3),
         "sem_score": round(float(sem_scores[i]), 3),
         "score": round(float(combined[i]), 3)}
        for i in ranked_idx
    ]


А.3. Метрики оцінювання (bookflix/ml/evaluation.py)

import math
import numpy as np
from sklearn.metrics import mean_squared_error

RELEVANCE_THRESHOLD = 7


def compute_rmse(actuals, predictions):
    return float(np.sqrt(mean_squared_error(actuals, predictions)))


def _dcg_at_k(relevances, k):
    relevances = relevances[:k]
    return sum(rel / math.log2(rank + 2)
               for rank, rel in enumerate(relevances))


def compute_precision_at_k(user_recommendations, user_relevant_items, k=10):
    scores = []
    for uid, recs in user_recommendations.items():
        relevant = user_relevant_items.get(uid, set())
        if not relevant:
            continue
        hits = sum(1 for isbn in recs[:k] if isbn in relevant)
        scores.append(hits / k)
    return float(np.mean(scores)) if scores else 0.0


def compute_ndcg_at_k(user_recommendations, user_relevant_items, k=10):
    scores = []
    for uid, recs in user_recommendations.items():
        relevant = user_relevant_items.get(uid, set())
        if not relevant:
            continue
        gains = [1 if isbn in relevant else 0 for isbn in recs[:k]]
        dcg = _dcg_at_k(gains, k)
        ideal = _dcg_at_k([1] * min(len(relevant), k), k)
        if ideal > 0:
            scores.append(dcg / ideal)
    return float(np.mean(scores)) if scores else 0.0


А.4. Корекція рейтингів VADER (bookflix/ml/sentiment.py)

import pandas as pd

ALPHA = 0.5


def _vader_compound(text: str) -> float:
    try:
        from vaderSentiment.vaderSentiment import SentimentIntensityAnalyzer
        _vader_compound._analyzer = getattr(
            _vader_compound, "_analyzer", SentimentIntensityAnalyzer())
        return _vader_compound._analyzer.polarity_scores(str(text))["compound"]
    except ImportError:
        pass
    try:
        from textblob import TextBlob
        return float(TextBlob(str(text)).sentiment.polarity)
    except ImportError:
        pass
    return 0.0


def build_book_sentiment_map(reviews_df: pd.DataFrame) -> dict:
    isbn_col = "isbn" if "isbn" in reviews_df.columns else "Book-ISBN"
    text_col = next(
        (c for c in ("ReviewContent", "review_text", "text")
         if c in reviews_df.columns), None)
    if text_col is None:
        return {}
    reviews_df = reviews_df.dropna(subset=[isbn_col, text_col]).copy()
    reviews_df["_compound"] = reviews_df[text_col].apply(_vader_compound)
    return reviews_df.groupby(isbn_col)["_compound"].mean().to_dict()


def apply_sentiment_correction(ratings_df, sentiment_map, alpha=ALPHA):
    ratings_df = ratings_df.copy()
    ratings_df["sentiment_score"] = (
        ratings_df["book__isbn"].map(sentiment_map).fillna(0.0))
    ratings_df["adjusted_rating"] = (
        ratings_df["book_rating"] + alpha * ratings_df["sentiment_score"]
    ).clip(1, 10)
    return ratings_df

    def __str__(self):
        return str(self.title)

    user_id = models.IntegerField(unique=True, primary_key=True)
    location = models.TextField()
    age = models.IntegerField(null=True, blank=True)

    def __str__(self):
        return f'User {self.user_id} from {self.location}'


class Rating(models.Model):
    user = models.ForeignKey(User, on_delete=models.CASCADE)
    book = models.ForeignKey(Book, on_delete=models.CASCADE, db_index=True)
    book_rating = models.IntegerField()

    def __str__(self):
        return (f'User {self.user.user_id} rated '
                f'{self.book.isbn} with {self.book_rating}')
